# Database
Obiettivi del Progetto

Modellazione concettuale: definire un Entity-Relationship Diagram (ERD) che rappresenti le entità principali (Veicolo, Cliente, Vendita, Modello) e le loro relazioni.

Normalizzazione: applicare le regole di normalizzazione fino alla terza forma normale (3NF) per garantire l'integrità e ridurre la ridondanza dei dati.

Implementazione: creare uno schema SQL su un DBMS (MySQL) con tabelle, vincoli di chiave primaria/esterna, check e trigger.

Popolamento dati: caricare dati sintetici e/o reali su modelli Tesla, listini prezzi, registrazioni clienti e cronologia vendite.

Interrogazioni avanzate: realizzare query SQL complesse per reportistica, analisi delle vendite e statistiche sui clienti.
